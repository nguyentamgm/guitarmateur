# Plan 05 — UI (`src/ui/`)

## Goals

Build the product UI on top of the engines. **Look and feel comes from the mockup**
([`Pentatonic Practice.dc.html`](../../Pentatonic%20Practice.dc.html)): dark theme, accent
`#c3f04b`, JetBrains Mono for labels/numbers/tab, panel-card rhythm, the 3-step page. **Behavior
comes from these plans**, not from the mockup's script. Open the mockup in a browser for visual
reference; never port its logic.

Recurring patterns to extract as small components/CSS classes (the mockup repeats these inline):

18px padding), `PillButton` (selected = accent bg/dark text, unselected = bordered transparent),
`TextButton` (underlined muted). Typography: system sans body; mono for kickers, fret numbers,
string labels, chip indices, tab.

## App shell (`App.tsx`)

Max-width 1080px container. Header (kicker "FRETBOARD TRAINER", h1 "Pentatonic Practice",
subtitle + "no audio — use your own backing track" note, copy per mockup — drop/replace the
no-audio note in M5). Then sections 1–3. The shell is a thin wrapper so a router + nav can be
added later without restructuring; keep page content in `PentatonicPracticePage.tsx` from the
start.

State comes from plan 06's provider: `const [state, dispatch] = useAppState()` — sections are
otherwise presentational.

## SVG renderers (pure presentational; typed props; no state access)

### `FretboardDiagram.tsx`
Props: `{ box: Box; mini?: boolean; highlight?: { chord: Chord; targetRole: ToneRole };
landing?: { string; fret }; labels?: 'names' | 'none' }`.
- Geometry per mockup: full `rowGap 24 / colW 44`, mini `9 / 15`; fret-marker dots (3 5 7 9 12 15
  17, double at 12), mono string letters + fret numbers in full mode, 3px nut line when
  `minFret === 0`. Full mode renders natural width inside an `overflow-x:auto` wrapper; mini
  scales to card width.
- Note styling by state: **chord target role** = solid accent fill + dark text; **other chord
  tones** = accent ring + 0.22 accent fill; **tonic** = 2.4px accent ring; **scale note** = faint
  ring, muted text; **decoration (♭5)** = dashed faint ring. `landing` note gets the dashed accent
  halo (the mockup's "start ring", reinterpreted: it marks where the lick *lands*).
- Note labels show spelled names (`B♭`) from `FretNote.pitch`; full mode may optionally label
  chord tones with roles (R/3/5/7) when `highlight` is set — small mono superscript, matches the
  "target the 3rd" feature.

### `TabStaff.tsx`
Props: `{ lick: Lick }`. Extends the mockup's look with rhythm + techniques:
- Horizontal placement **proportional to `startBeat`**; beat ticks/numbers along the top (mono).
- Fret number on the string line in a card-bg rounded rect (mockup style).
- Duration row under the staff: compact mono glyphs per note (`♩`, `♪`, `♬`-style or `1 1/2 1/4`
  text — pick one, keep it legible at 11px; document the choice in the component).
- Technique glyphs between connected notes, standard tab notation: `h`, `p`, `/` (slide up),
  `\` (down), `b½`/`b` (bends) rendered above the pair with a small slur arc.
- Empty lick: centered em-dash.

### `Legend.tsx`
Swatch circles matching the note styles: scale note, tonic, chord tone, target (role-aware label:
"target · 3rd"), landing, decoration (♭5). Only show entries relevant to the section.

## Sections

### 1 — Scale & Position (`ScalePositionSection.tsx`)
- **Key picker**: 12 pills from the music core's tonic list (A first), spelled labels (`B♭` not
  `A♯`). **Scale picker**: pills driven by the scale registry — rendering the registry means a new
  scale appears in the UI by itself.
- Changing key or scale ⇒ dispatch resets progression to `defaultProgression` + reseeds licks
  (plan 06 owns the semantics).
- **Position cards**: horizontal scroll row, minFret-sorted; mini fretboard, "Position N", mono
  fret range, "REC" accent badge from `recommendedPosition`. Selection = accent border + tint.
  Toggle semantics (max 2, adjacent-only combine, min 1) come from `areAdjacent` — UI contains no
  adjacency logic.
- **Merged diagram**: title "B♭ Minor Pentatonic — frets 6–9 (combined)", legend, full diagram.

### 2 — Chord Progression (`ProgressionSection.tsx`)
- **Suggestion chips** from `suggestedChords(key)`: "+ Cm" with a muted mono roman-numeral
  sub-label ("iv"). Tap = append chord (its lick seed is minted by the reducer).
- **Advanced adder** (collapsed by default): root picker (12 spelled pills) + quality pills from
  the chord-quality registry + live-preview "Add B♭m7" accent button.
- **Progression chips**: numbered, HTML5 drag-to-reorder, "×" remove; "Reset to default" /
  "Clear all" text buttons; empty-state line. (Per-chord bar counts arrive in M5 — leave room in
  the chip layout, don't build it.)

### 3 — Practice (`PracticeSection.tsx`)
- **Controls bar**: **Level** segmented control 1–5 (replaces the mockup's notes-per-chord
  slider — note count now emerges from rhythm; levels 4–5 disabled with a "soon" tooltip until
  M4) · **Target** segmented control R / 3rd / 5th · **Land on next chord** toggle (M4; hidden in
  M3) · "↻ Regenerate" button (reseeds all licks).
- **Cards grid** (`auto-fill minmax(320px, 1fr)`): chord label + roman numeral, accent
  "target · 3rd (E♭)" badge, full fretboard with chord highlighting + landing halo, "LICK" kicker,
  `TabStaff`, and a small per-card ↻ (reseed just this card).
- Legend above the grid; dashed-panel empty state.

## Responsiveness & a11y (reasonable effort, no framework)

- 375px width: no page-level horizontal scroll; wide SVGs scroll inside their own wrappers.
- Real `<button>`s everywhere; `aria-pressed` on toggles/pills; `aria-label` on icon-only buttons
  (chip ×, per-card ↻); one `<title>` per SVG ("B♭ minor pentatonic, frets 6–9"); visible accent
  focus outline (`:focus-visible`).

## Manual verification checklist (per milestone; replaces E2E)

M2: key/scale/position interactions; spelled names on the diagram (check F minor shows B♭/D♭);
combined boxes; REC badge; mobile width.
M3: full loop — defaults on first load (A minor pent, Am–G–F–G, level 2); suggestions with
numerals; advanced adder; drag reorder keeps cards in sync; level changes rhythm density
visibly; every card's landing note matches the target control; tab shows rhythm; reload restores
everything; regenerate (global + per-card) changes licks.
M4: technique glyphs render; land-on-next mode targets the following chord.
